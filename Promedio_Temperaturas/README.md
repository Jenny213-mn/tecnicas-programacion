# Promedio de Temperaturas

Este repositorio contiene dos programas:

## Programación Tradicional
Archivo: clima_tradicional.py  
Calcula el promedio semanal de temperaturas usando funciones.

## Programación Orientada a Objetos (POO)
Archivo: clima_poo.py  
Calcula el promedio semanal aplicando clases, objetos y encapsulamiento.

## Comparación entre Programación Tradicional y Programación Orientada a Objetos

La Programación Tradicional se basa en el uso de funciones y una estructura secuencial,
lo que permite resolver problemas simples de manera directa y comprensible.
En este enfoque, los datos y las funciones están separados, lo cual puede dificultar
el mantenimiento del código cuando el programa crece.

Por otro lado, la Programación Orientada a Objetos (POO) organiza el programa en clases
y objetos, permitiendo encapsular los datos y las operaciones relacionadas.
Esto mejora la modularidad, la reutilización del código y facilita el mantenimiento.
En este proyecto, la POO permitió representar la información climática mediante una clase,
aplicando el encapsulamiento como principio fundamental.
En conclusión, ambos enfoques cumplen el objetivo de calcular el promedio semanal de
temperaturas, pero la Programación Orientada a Objetos ofrece una estructura más escalable
y organizada para proyectos de mayor tamaño.
